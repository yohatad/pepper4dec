   **Investigated and deferred (2026-07-08):** every Image-consuming node here — `face_detection`, `person_detection`, `overt_attention_saliency`, `overt_attention_visualization` — subscribes directly and independently to the same external camera-driver topic (`topics_config_.image.pepper`/`.realsense`); none of them receive an `Image` from another in-repo node (`face_detection`'s only cross-node subscription is `person_detection`'s structured `PersonDetection` message, not an image). Revisit only if `realsense2_camera`'s composability is confirmed useful and the RealSense path becomes primary.
